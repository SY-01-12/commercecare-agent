## 阶段 1：基础项目复现 🚧

> 分支：`feat/base-reproduction`

**目标**：基于上游项目在本地跑通可运行的基础版本。

| 任务 | 说明 | 状态 |
|------|------|------|
| 1.1 初始化项目结构 | README, .gitignore, .env.example 等 | ✅ 已完成 |
| 1.2 搭建 Python 后端 | FastAPI + OpenAI Agents SDK + ChatKit | ⬜ 待开始 |
| 1.3 搭建 Next.js 前端 | ChatKit React 集成，基础聊天 UI | ⬜ 待开始 |
| 1.4 复现 Agent 编排 | Triage + Specialist + Handoff 模式 | ⬜ 待开始 |
| 1.5 复现 Guardrail | Relevance + Jailbreak 输入检查 | ⬜ 待开始 |
| 1.6 端到端验证 | 完整 Demo Flow 可运行 | ⬜ 待开始 |

**交付物**：可运行的基础客服系统（航旅场景），验证技术栈可行性。

> 分支：`feat/ecommerce-scenario`

**目标**：将航旅场景替换为电商售后客服场景。

| 任务 | 说明 | 状态 |
|------|------|------|
| 2.1 重构数据模型 | `AirlineAgentContext` → `CommerceCareAgentContext` | ⬜ 待开始 |
| 2.2 设计 Agent 角色 | 订单、物流、退款、FAQ、人工转接 Agent | ⬜ 待开始 |
| 2.3 编写电商 Mock 数据 | 订单、物流、退款 Mock 数据集 | ⬜ 待开始 |
| 2.4 重构 Guardrail | 相关性判定标准改为电商售后范围 | ⬜ 待开始 |
| 2.5 更新前端文案 | 界面文案改为电商售后场景 | ⬜ 待开始 |

**交付物**：电商售后客服基础版（Mock 数据驱动）。

## 阶段 3：RAG 企业知识库

> 分支：`feat/rag-knowledge-base`

**目标**：集成向量检索，实现基于企业知识库的智能问答。

| 任务 | 说明 | 状态 |
|------|------|------|
| 3.1 搭建向量数据库 | ChromaDB / FAISS 本地部署 | ⬜ 待开始 |
| 3.2 文档导入管道 | PDF / Markdown / 网页 → 分块 → 向量化 | ⬜ 待开始 |
| 3.3 RAG 检索工具 | 基于用户问题的语义检索工具 | ⬜ 待开始 |
| 3.4 集成到 FAQ Agent | FAQ Agent 使用 RAG 替代 if/else | ⬜ 待开始 |
| 3.5 检索质量评估 | 召回率 / 精确率 / 人工评估 | ⬜ 待开始 |

**交付物**：FAQ Agent 基于真实企业知识库回答。

---

## 阶段 4：订单、物流、退款工具调用

> 分支：`feat/tool-integration`

**目标**：实现可调用的业务工具。

| 任务 | 说明 | 状态 |
|------|------|------|
| 4.1 订单查询工具 | 按订单号/手机号查询订单详情 | ⬜ 待开始 |
| 4.2 物流追踪工具 | 查询物流状态、异常信息 | ⬜ 待开始 |
| 4.3 退款处理工具 | 发起退款、查询退款进度 | ⬜ 待开始 |
| 4.4 退货处理工具 | 生成退货单、退货地址查询 | ⬜ 待开始 |
| 4.5 数据持久化 | SQLite 存储订单/会话/日志 | ⬜ 待开始 |

**交付物**：Agent 可调用工具处理真实业务。

## 阶段 5：人工客服转接与工单

> 分支：`feat/human-handoff`

**目标**：实现复杂问题的人工转接机制。

| 任务 | 说明 | 状态 |
|------|------|------|
| 5.1 转接判定逻辑 | 置信度阈值 / 关键词 / 用户请求 | ⬜ 待开始 |
| 5.2 工单系统 | 创建、分配、跟踪、关闭工单 | ⬜ 待开始 |
| 5.3 转接通知 | Webhook / 消息队列通知人工客服 | ⬜ 待开始 |
| 5.4 会话交接 | 上下文摘要传递给人工客服 | ⬜ 待开始 |
| 5.5 转接 UI | 前端转接状态展示和处理 | ⬜ 待开始 |

**交付物**：Agent 可将复杂问题无缝转接人工客服。
